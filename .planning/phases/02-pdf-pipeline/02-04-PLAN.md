<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| buildCandidateContext() → index.ts | CandidateContext.cvText is attacker-influenced content (from PDF); only used as a string, never executed |
| candidate loop → logDecision() | needsReviewReason value from CandidateContext is logged to stdout as structured JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Information Disclosure | cvText logged to stdout | mitigate | cvText is NOT logged in logDecision() for the needsReview path (only ctx.needsReviewReason is in reasons[]). For the pass path, only the placeholder string 'CV extracted; pending Phase 3 agent evaluation' is logged — not the CV text. GDPR compliance: CV PII is held in memory only, never persisted to stdout JSON stream. Verify: no `ctx.cvText` appears in logDecision() calls in index.ts. |
| T-02-04-02 | Denial of Service | buildCandidateContext() blocks per candidate, no timeout | accept | Per-candidate call is already isolated by try/catch (SAFE-01). Node.js fetch has no default timeout; BambooHR PDF download could hang. LOW risk in practice — BambooHR API is the controlled environment. Phase 4 may add fetch timeouts. |
| T-02-04-03 | Tampering | needsReviewReason string injected into JSON log output | accept | NeedsReviewReason is a string literal union type — values are hardcoded in extract-cv.ts. TypeScript compiler prevents arbitrary strings at compile time. Runtime: the value originates from extract-cv.ts code paths, not from external input. |
</threat_model>
