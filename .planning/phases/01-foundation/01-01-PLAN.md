<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → process | config.yaml is read from disk and parsed by js-yaml; malformed or adversarial YAML enters here |
| OS environment → process | credentials (BAMBOOHR_API_KEY, BAMBOOHR_SUBDOMAIN) arrive via env vars; leakage risk is at process boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | config.yaml parsing | mitigate | Use `yaml.load()` with default schema (safe in js-yaml v4); do NOT use `{ schema: DEFAULT_SCHEMA }` with untrusted YAML types |
| T-01-02 | Information Disclosure | .env credentials in git | mitigate | .gitignore includes `.env`; .env.example committed with empty values only; CI must never receive real creds |
| T-01-03 | Information Disclosure | BAMBOOHR_API_KEY in logs | mitigate | Task actions explicitly prohibit logging env var values; credential values must never appear in console output |
| T-01-04 | Tampering | package.json supply chain | accept | npm install uses package-lock.json pinning; low-value target (no production deploy in Phase 1) |
</threat_model>
